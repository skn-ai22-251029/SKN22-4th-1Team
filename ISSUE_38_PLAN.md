# Issue #38 작업 계획

이 문서는 [Issue #38](https://github.com/skn-ai22-251029/SKN22-4th-1Team/issues/38) 구현을 위한 실행 계획입니다.

## 1. 배경
- 회원가입과 회원정보 수정의 입력 규칙이 달라 약물/기저질환/알레르기 데이터 정합성이 깨질 수 있음
- 특히 회원정보 수정(`/user/profile/`)에서 자유 타이핑 저장이 가능해 표준화 실패 가능성이 큼
- KCD 참조 소스(table/column) 기준이 명확하지 않아 환경별 동작 차이가 발생할 수 있음
- 일부 화면에서 한글 문구 인코딩 깨짐이 관찰됨

## 2. 현재 문제
- 회원정보 수정에서 복용중인약 임의 문자열 저장 가능
- 기저질환/알레르기 입력이 KCD 선택형으로 강제되지 않음
- KCD 소스 테이블/컬럼 운영 기준 미고정
- 화면 문구 일부 인코딩 깨짐

## 3. 재현 절차
1. 로그인 후 `/user/profile/` 이동
2. 복용중인약/알레르기/기저질환에 임의 문자열 입력
3. 저장 후 값이 그대로 반영되는지 확인

## 4. 목표
- 가입/수정 모두 동일한 입력-검증-저장 규칙으로 통일
- 질환/알레르기 데이터를 `질병명 [코드]` 형태로 표준화
- 약물은 검색 선택값만 저장되도록 강제

## 5. 작업 범위
- In Scope
- 회원정보 수정 화면 약물 자동완성 선택형 UI 적용 (`/drug/search/`)
- 회원정보 수정 화면 기저질환/알레르기 KCD 자동완성 태그 UI 적용 (`/user/kcd/search/`)
- 프론트/백엔드 이중 검증으로 약물 자유 입력 차단
- 질환/알레르기 저장 포맷 `질병명 [코드]` 통일
- KCD 소스 테이블/컬럼 운영 기준 고정 (`kcd_info` 또는 단일 표준)
- KCD 미일치/모호매칭/소스없음 오류 메시지 정리
- 회원가입/회원정보수정 인코딩(UTF-8) 깨짐 정리
- 가입/수정/에러 케이스 회귀 테스트
- Out of Scope
- 이슈 #38과 무관한 대규모 리팩터링

## 6. 구현 계획
1. UI 규칙 통일
- 회원가입 페이지와 동일한 선택형 컴포넌트를 회원정보 수정에도 재사용
- 자유 텍스트 입력 필드를 선택형 태그 입력으로 대체

2. API/백엔드 검증 강제
- 약물: 선택된 항목 ID/정규화명 외 저장 거부
- 질환/알레르기: KCD 코드 매핑 성공 항목만 저장 허용
- 실패 시 원인 기반 메시지 반환 (미일치/모호/소스없음)

3. KCD 소스 기준 고정
- 서비스 레이어에서 KCD 참조 테이블/컬럼을 단일 기준으로 통일
- 소스 미구성 시 사용자에게 명확한 오류 노출

4. 인코딩 정리
- 템플릿/응답/저장 경로 UTF-8 기준 점검 및 깨짐 문자열 치환

5. 회귀 검증
- 가입과 수정의 저장 포맷/검증 동작 일치 여부 확인
- 에러 응답 문구와 UI 안내 문구 일치 여부 확인

## 9. 수용 기준 (Acceptance Criteria)
- 가입/수정 모두 임의 문자열 약물 저장 불가
- 가입/수정 모두 KCD 매핑 성공 항목만 질환/알레르기 저장
- 저장값에 코드 포함 (`고혈압 [I10]` 형식)
- KCD 소스 미구성 시 원인 기반 오류 메시지 노출
- 화면 한글 라벨/문구 정상 표시

## 10. 테스트 시나리오
1. 정상
- 약물 검색 선택 후 저장 성공
- KCD 검색 선택 후 저장 성공
- 저장값 형식이 `이름 [코드]`인지 확인

2. 예외
- 약물 임의 문자열 입력 후 저장 시 차단
- KCD 미일치 문자열 입력 시 차단
- KCD 모호 매칭 입력 시 명확한 안내 메시지 확인
- KCD 소스 비활성/미연결 상태에서 원인 메시지 확인

3. 회귀
- 회원가입과 회원정보 수정의 저장 결과 포맷 동일성 확인
- 기존 프로필 조회/표시 기능 영향 없음 확인

3. Day 3: 회귀 테스트 + 리뷰 반영 + 최종 정리
