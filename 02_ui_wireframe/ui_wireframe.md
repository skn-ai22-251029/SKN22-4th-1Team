# 02. 화면 설계서 (UI Wireframe)

## 기준 정보

- **기준 브랜치**: `develop`
- **기준 서버**: Django ASGI + Uvicorn
- **기준 템플릿 위치**:
  - `skn22_4th_prj/templates/index.html` (메인)
  - `skn22_4th_prj/templates/login.html` (로그인)
  - `skn22_4th_prj/templates/register.html` (회원가입)
  - `skn22_4th_prj/templates/profile.html` (프로필)
  - `skn22_4th_prj/templates/search_result.html` (통합 검색 결과)
  - `skn22_4th_prj/templates/symptom_result.html` (증상 기반 결과)
  - `skn22_4th_prj/templates/symptom_products_page.html` (제품 추천)
  - `skn22_4th_prj/templates/error.html` (오류 페이지)

## 1. 공통 레이아웃 (Common Layout)

### 1.1 네비게이션 & 헤더
- **Top Header**
  - 로고 및 서비스 명 (클릭 시 Home `/` 이동)

## 2. 화면 상세 설계

### 2.1 메인 페이지 (`index.html` - `/`)

**목적**: OTC 복약 안전 정보 제공 서비스의 진입점. 검색과 약국 탐색 기능 제공

**구성 요소**:

#### 2.1.1 Hero Section
- **제목**: "OTC 복약 안전 가이드"
- **부제**: "증상, 제품명, 또는 주변 약국을 검색하세요"
- **검색 입력 폼**:
  - 텍스트 입력 필드 (placeholder: "증상/제품명 입력...")
  - 검색 버튼
- **추천 검색어**: 칩(Chip) 버튼 형태 (예: "감기약", "소화제", "감기", "소화불량")

#### 2.1.2 Feature Cards (3열 레이아웃)
- **카드 1**: 🏥 증상별 안내
  - 복용 가능 성분 정보
  - DUR 검사 및 주의사항
- **카드 2**: 💊 제품 검색
  - OTC 제품 추천
  - DUR 캐테고리별 필터링
- **카드 3**: 🗺️ 약국 찾기
  - 근처 약국 위치 검색
  - 운영시간 및 연락처 조회

#### 2.1.3 약국 탐색 섹션
- **위치 제어 (Location Control)**:
  - "[내 위치 찾기]" 버튼 (Geolocation API)
  - 상태 메시지 표시 (예: "위치를 허용하세요", 현재 좌표 표시)
- **지도 영역** (높이: 400px)
  - Leaflet 또는 Google Maps 라이브러리 사용
  - 현재 위치: 🔴 빨간 마커
  - 약국 위치: 📍 파란/초록 마커
  - **상호작용**:
    - 마우스 호버: 약국 간단 정보 (이름, 전화) 툴팁
    - 마우스 클릭: 상세 팝업 (주소, 전화, 운영시간)
    - 지도 이동 완료(moveend): 현재 Viewport 기준 약국 재조회
- **약국 목록 영역** (하단, 스크롤 지원)
  - 약국명 / 주소
  - 전화번호 / 운영시간 / 영업상태
  - 목록 항목 클릭 시 지도 마커에 하이라이트

  - 비밀번호 입력 필드
  - "[로그인]" 버튼
  - "[비밀번호 찾기]" 링크

### 2.3 회원가입 페이지 (`register.html` - `/register/`)

**목적**: 신규 사용자 계정 생성

**구성 요소**:
- **회원가입 폼**:
  - 이메일 입력 필드 (중복 확인)
  - 사용자명 입력 필드
  - 비밀번호 입력 필드
  - 비밀번호 확인 입력 필드
  - "[가입하기]" 버튼
- **로그인 유도**: "이미 계정이 있으신가요? [로그인]" 텍스트 링크

---

### 2.4 프로필 페이지 (`profile.html` - `/profile/`)

**목적**: 사용자 건강 정보 관리 및 맞춤형 검색 결과 제공

**구성 요소**:
- **프로필 섹션**:
  - 사용자명 / 이메일 표시
- **건강 정보 입력 (선택사항)**:
  - **현재 복용 약물**: 텍스트 입력 (쉼표 분리)
  - **알레르기 정보**: 텍스트 입력
  - **기저질환**: 선택 체크박스 또는 입력 필드
  - **임신 여부**: 라디오 버튼 (예/아니오)
- **저장 버튼**: 변경사항 저장
- **로그아웃 버튼**: 프로필 페이지 하단

---

### 2.5 통합 검색 결과 페이지 (`search_result.html` - `/smart-search/?q={query}`)

**목적**: 사용자 질의를 증상/제품으로 분류하고 관련 정보 제시

**구성 요소**:

#### 2.5.1 상단 네비게이션
- "[← 뒤로가기]" 버튼
- 검색 키워드 표시: "검색 결과: \"{query}\""
- **탭 메뉴**: "증상 결과" | "제품 결과"

#### 2.5.2 증상 결과 탭
- **상담 메모 박스**:
  - 영어로 작성된 의사/약사 상담용 메모
  - 구성:
    - Chief Complaint: 사용자 증상 영문 번역
    - Profile: 나이, 성별, 복용약, 알레르기, 기저질환, 임신 여부
    - Recommended OTC Ingredients: 추천 성분 목록
  - 복사 버튼 (📋 메모 복사)
- **복용 가능 성분 카드** (DUR 통과):
  - 각 성분별 카드:
    - 성분명 (한글) + (영문)
    - 용도 및 효능
    - DUR 안내 상태: ✅ OK 또는 ⚠️ 주의 (사유 표시)
    - 프로필 기반 주의 메시지 (있을 경우)
    - "[제품 보기]" 버튼 → 클릭 시 `/smart-search-products/` 이동

#### 2.5.3 제품 결과 탭
- 검색 키워드와 매칭되는 제품명 목록
- 각 제품 상품:
  - 제품명 / 제조사
  - 효능 요약
  - 주성분 표시
  - "[상세 정보]" 링크 (외부 페이지 또는 모달)

---

### 2.6 성분별 제품 추천 페이지 (`symptom_products_page.html` - `/smart-search-products/`)

**목적**: 특정 성분에 대한 OTC 제품 목록 비동기 조회 (검색 결과 후속 페이지)

**구성 요소**:

#### 2.6.1 상단 헤더
- "[← 뒤로가기]" 버튼
- 성분명 표시: "{성분명} (Ingredient) 관련 OTC 제품"
- 비동기 로딩 상태 표시

#### 2.6.2 제품 리스트 영역
- **로딩 상태**:
  - 스켈레톤 로더 또는 "[로딩 중...]" 메시지
- **데이터 로드 후 제품 카드 리스트**:
  - 각 제품 카드 구성:
    - 제품명
    - 제조사
    - 효능 요약
    - 💊 주성분 및 용량
    - 📋 주의사항 (최대 일일 복용량 등)
    - 🔗 상세정보 링크 (FDA 또는 공식 페이지)
    - DUR 안내: 프로필 기반 안전성 평가 (✅ OK 또는 ⚠️ 주의)
    - "[상세 정보]" 버튼 / "[약국 찾기]" 버튼
- **오류 상태**:
  - API 실패 시: "제품 정보를 불러올 수 없습니다. 나중에 다시 시도하세요."

### 2.7 오류 페이지 (`error.html` - `/error/`)

**목적**: 시스템 오류 및 API 실패 상황 사용자 안내

**구성 요소**:
- **오류 타입별 메시지**:

| 오류 상황 | 메시지 |

- **재시도/홈 이동 버튼**

## 3. 사용자 흐름도 (User Flow)

### 3.1 증상 기반 검색 플로우
1. 메인 페이지 (`/`)
2. (미로그인 시) 로그인/회원가입 페이지 (`/login/` 또는 `/register/`)
3. (선택) 프로필 정보 입력 (`/profile/`)
4. 검색창에 증상 입력 → 검색 버튼 클릭
5. 통합 검색 결과 페이지 (`/smart-search/?q={query}`)
6. 증상 탭 전환 → 상담 메모 + 성분 카드 노출
7. "[제품 보기]" 클릭 → 제품 추천 페이지 (`/smart-search-products/`)
8. 제품 정보 확인 및 "[약국 찾기]" 클릭 (선택)

### 3.2 약국 검색 플로우
1. 메인 페이지 (`/`)
2. "[내 위치 찾기]" 클릭 → Geolocation 권한 요청
3. 좌표 획득 → 지도 + 약국 목록 렌더링
4. 지도 호버: 약국 간단 정보 표시
5. 지도 클릭: 상세 팝업 (주소, 전화, 운영시간)
6. 지도 이동 → moveend 이벤트 → 현재 Viewport 기준 약국 재조회
7. 목록 항목 클릭 → 지도 마커 하이라이트

### 3.3 인증 플로우
1. 메인 페이지 (미로그인) → "[로그인]" 또는 "[회원가입]" 클릭
2. 로그인/회원가입 페이지
3. 계정 생성/로그인 성공 → 세션 저장
4. 메인 페이지 리다이렉트
5. (선택) "[프로필]" 링크 → 프로필 설정 (약물/알레르기/기저질환/임신)

## 4. 상호작용 & 상태 관리 (Interaction & State)

### 4.1 검색 상호작용
- **입력 필드**: 
  - 텍스트 입력 시 자동 제안 (선택)
  - Enter 키 또는 "검색" 버튼으로 제출
- **로딩 상태**: 제출 후 결과 반환까지 버튼 비활성화 + 로딩 스피너 표시

### 4.2 지도 상호작용
- **호버(Hover)**: 마커 위 마우스 올림 → 약국명/전화 툴팁
- **클릭(Click)**: 마커 클릭 → 팝업 표시 (주소, 전화, 운영시간)
- **지도 이동(Moveend)**: 사용자가 지도를 드래그/줌 완료 후 현재 영역 기준 약국 재조회
- **필터링**: Pet/Vet 카테고리 제외 자동 필터링

### 4.3 세션 상태 관리
- **미로그인 상태**: 모든 페이지 접근 가능 (프로필 기반 정보 미제공)
- **로그인 상태**: 
  - 프로필 정보 저장 및 활용
  - 검색 결과에 사용자 맞춤 정보 반영
  - 헤더에 "[프로필]" 및 "[로그아웃]" 표시

### 4.4 오류 상태 처리
- **네트워크 실패**: 재시도 버튼 제공
- **권한 거부**: 사용자 브라우저 설정 안내
- **결과 없음**: 다른 검색어 제안 또는 메인 페이지 복귀 옵션

## 5. API 엔드포인트 (Frontend 관점)

| 엔드포인트 | Method | 목적 | 요청 파라미터 | 응답 |
|-----------|--------|------|-------------|------|
| `/api/search/` | POST | 통합 검색 분류 | `{ "q": "string" }` | `{ "type": "symptom\|product", "data": {...} }` |
| `/api/symptom-products/` | GET | 성분별 제품 조회 | `component={name}` | Array of product objects |
| `/api/pharmacies/` | GET | 위치 기반 약국 조회 | `lat, lng, radius, limit` | Array of pharmacy objects |
| `/api/user-profile/` | GET/POST | 사용자 프로필 관리 | (인증 필요) | User profile object |
| `/smart-search/` | GET | 검색 결과 페이지 렌더 | `q={query}` | HTML |
| `/smart-search-products/` | GET | 제품 추천 페이지 렌더 | `component={name}` | HTML |

## 6. 반응형 디자인 (Responsive Design)

- **Desktop** (1200px+): 2~3열 레이아웃, 풀사이즈 지도
- **Tablet** (768px~1199px): 1~2열 레이아웃, 축소된 지도
- **Mobile** (< 768px): 1열 레이아웃, 수직 스택 방식, 지도 높이 축소

## 7. 추가 고려사항

### 7.1 접근성 (Accessibility)
- ARIA 라벨 제공
- 키보드 네비게이션 지원
- 색상 대비 WCAG 기준 충족

### 7.2 성능 (Performance)
- 초기 렌더: 메인 콘텐츠 우선 로드
- 비동기 작업: 제품 조회, 약국 조회 병렬 처리
- 이미지/스크립트: 지연 로딩(Lazy Loading) 적용

### 7.3 보안 (Security)
- 사용자 입력 XSS 방지 (마크다운 파싱 서니타이제이션)
- CSRF 토큰 사용
- API 요청 시 인증 헤더 전송

### 7.4 SEO (Search Engine Optimization)
- Meta 태그 (title, description, og:image)
- 시맨틱 HTML 구조
- 모바일 친화적 인덱싱
