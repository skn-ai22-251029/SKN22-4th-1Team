# 01. 요구사항 정의서 (Requirements Specification)

## 기준 정보

- 기준 브랜치: `develop`
- 기준 코드 위치: `C:\codes\SKN22-4th-1Team`

## 1) 프로젝트 개요

- 프로젝트명: Global Drug Safety Intelligence (GDSI)
- 목적: 증상/제품명 기반 OTC 복약 안전 안내 제공
- 핵심 차별점: 미국 FDA + 국내 DUR + 사용자 건강정보 반영

## 2) 기능 요구사항

### FR-01 통합 검색
- 사용자는 메인 페이지에서 증상 또는 제품명을 입력할 수 있어야 한다.
- 시스템은 질의를 분류하여 증상 플로우/제품 플로우로 처리해야 한다.

### FR-02 증상 기반 성분 안내
- 증상에 대한 복용 가능 성분을 제시해야 한다.
- 성분별 DUR 위험 유형 및 사유를 표시해야 한다.

### FR-03 제품 안내
- 복용 가능 성분에 대해 OTC 제품 후보를 비동기로 조회해야 한다.
- 제품명/제조사/효능 요약/주의사항을 제공해야 한다.

### FR-04 개인화 반영
- 사용자 프로필(복용약/알레르기/기저질환/임신여부)을 결과에 반영해야 한다.
- 의사/약사 상담용 메모를 영어로 생성해야 한다.

### FR-05 주변 약국 조회
- 사용자는 `내 위치 찾기` 후 주변 약국 조회가 가능해야 한다.
- 지도와 목록을 동시에 제공해야 한다.
- 지도 이동 시 현재 viewport 기준으로 약국을 재조회해야 한다.
- pet/vet 계열 장소는 제외해야 한다.

### FR-06 지도 상호작용
- 현재 위치는 빨간 마커로 표시해야 한다.
- 약국 핀 hover/click 시 상세 정보(주소/전화/운영시간)를 제공해야 한다.

## 3) 비기능 요구사항

- 성능: 초기 렌더는 빠르게, 추가 데이터는 비동기 로딩
- 안정성: 외부 API 실패 시 fallback 및 사용자 안내 메시지 제공
- 보안: API 키는 `.env`로 관리
- 접근성: 모바일/데스크톱 레이아웃 유지

## 4) 수용 기준

- AC-01: `/smart-search/` 결과 페이지 정상 렌더
- AC-02: 상담 메모에 영어 증상/영어 프로필 정보 출력
- AC-03: `/api/pharmacies/`로 지도+목록 표시
- AC-04: 지도 이동 후 목록 갱신
- AC-05: pet/vet 장소가 결과에서 제외
