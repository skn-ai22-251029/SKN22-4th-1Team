## 🎯 트리거 조건
안티그래비티 (또는 다른 AI 에이전트)는 다음 상황에서 이 Skill을 **자동으로 읽고 실행**해야 합니다:
1. 사용자가 "PR 올려줘", "풀 리퀘스트 만들어", "작업 다 했으니 병합 요청해줘" 등의 요청을 했을 때
2. **[auto-commit] 스킬의 실행이 완료된 직후**, 에이전트가 "바로 PR을 생성할까요?"라고 제안하고 사용자가 승인했을 때

## 🛠️ 실행 규칙 (User Confirmation Flow)

사용자가 PR 생성을 요청하면, **절대 바로 PR을 쏘지 말고**, 변경 사항을 분석해 먼저 사용자에게 확인을 받아야 합니다.

### 1️⃣ 정보 수집 및 초안 작성
1. **변경 사항 분석**: 현재 작업 브랜치의 커밋 내역(`git log`)이나 변경 파일 목록(`git diff`)을 분석합니다.
2. **PR 초안 생성**: 분석한 내용을 바탕으로 PR의 제목과 마크다운 본문 초안을 작성합니다.

### 2️⃣ 대상 브랜치 확인 및 유도성 질문
초안이 준비되면 로컬 또는 원격 브랜치 목록(`git branch -r`)을 훑어보고(`main`, `develop` 등), 사용자에게 다음과 같은 뉘앙스로 묻습니다:

> "✅ 현재 `[작업 브랜치명]` 브랜치에서 PR(Pull Request) 생성을 준비했습니다.
> 
> **[PR 타겟 브랜치 선택]**
> 병합할 대상(Base) 브랜치를 선택해 주세요! (발견된 주요 브랜치: `main`, `develop` 등)
> (선택하지 않으시면 자동으로 `develop` 또는 `main`으로 설정됩니다.)
> 
> **[PR 초안]**
> - **제목**: `feat: 카카오 로그인 기능 연동`
> - **본문**: ...
> - 관련 이슈: #123 (GitHub 이슈 오토메이션 스킬로 생성된 이슈 등)
> 
> 이 내용으로 `SKN22-4th-1Team` 리포지토리의 `[선택한 타겟 브랜치]` 브랜치로 PR을 올릴까요? 
> 내용이 괜찮으시면 **'진행해'**(또는 'develop에 머지해줘'), 수정/취소를 원하시면 **'하지 마'**라고 말씀해 주세요."

### 3️⃣ 사용자의 응답에 따른 처리
- **사용자가 긍정 대답을 한 경우 (Target 브랜치 지정 포함)**:
  - `gh` CLI 명령어를 사용하여 PR을 생성합니다.
  - 실행 대상: **`skn-ai22-251029/SKN22-4th-1Team`**
  - 사용자가 명시한 Base 브랜치를 `--base` 옵션으로 반드시 넘겨줍니다.
  - 명령어 예시: `gh pr create --repo skn-ai22-251029/SKN22-4th-1Team --base [선택된브랜치] --title "[PR 제목]" --body "[본문 내용]"`
  - 생성 성공 시, 생성된 PR의 링크나 상태를 사용자에게 안내합니다.
- **사용자가 "하지 마", "아니", "추가로 고칠게" 등의 대답을 한 경우**:
  - **절대 PR 생성을 진행하지 않습니다.**
  - 사용자의 추가 지시(내용 수정 등)를 대기합니다.

### 📝 PR 작성 규칙
- **언어**: PR의 제목과 본문은 모두 **'한국어'**로 자연스럽게 작성합니다.
- **제목 컨벤션**: 커밋 메시지와 동일한 Conventional Commits 접두어(`feat:`, `fix:` 등)를 가급적 준수합니다.
- **포함 내용**: `What (무엇을 해결/추가했는지)`, `Why (왜 했는지)`, `관련 이슈 번호`가 들어가면 가장 좋습니다.
