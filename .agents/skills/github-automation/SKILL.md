## 🎯 트리거 조건
안티그래비티 (또는 다른 AI 에이전트)는 다음 상황에서 이 Skill을 **자동으로 읽고 실행**해야 합니다:
1. 사용자가 버그 수정이나 새 기능 추가에 대한 대화를 나누고, **에이전트가 그 원인이나 앞으로의 수정 계획을 명확히 파악(합의)한 바로 그 시점**.
   - **⚠️ 중요**: 코드를 본격적으로 수정하기 전, 에이전트 선에서 먼저 "원인과 수정 계획이 명확해졌습니다. 본격적인 작업에 앞서 이 내용을 바탕으로 GitHub 이슈를 생성하고 프로젝트 보드에 등록할까요?" 라고 **사용자에게 승인을 요청**해야 합니다.
2. 사용자가 명시적으로 "이슈 만들어줘" 또는 "/이슈등록"을 요청했을 때

## 🛠️ 실행 지침 (이슈 생성 및 Projects 등록)

AI 에이전트는 아래의 우선순위와 규칙에 따라 작업을 수행합니다.

### 🛑 [필수] 0순위: 보안 및 민감 정보 검증 (Security Check)
이슈나 PR을 생성하기 전, **반드시 작성될 본문과 제목을 검증**합니다.
- 본문에 API 키(`sk-...`, `ghp_...` 등), 비밀번호, 데이터베이스 접근 정보, 개인정보가 절대 포함되지 않도록 확인합니다.
- 만약 코드 조각에 민감 정보가 포함되어 있다면, 이를 `***` 로 마스킹(Masking) 처리하거나 해당 부분을 안전한 더미 데이터(`YOUR_API_KEY`)로 치환한 뒤 업로드합니다.
- 이 검증 과정을 통과하지 못하면 작업을 중단하고 사용자에게 경고해야 합니다.

### 1순위: GitHub CLI (`gh`) 사용
터미널에서 `gh` 명령어를 최우선으로 실행하여 처리합니다.

### 2순위: GitHub MCP Server 사용
`gh` 명령어 사용이 불가능하거나 에러가 날 경우, `github-mcp-server` MCP 도구를 사용합니다.
- 사용 도구: `mcp_github-mcp-server_create_issue` 등

---

### 📋 이슈 생성 규칙
이슈 생성 시 다음 **필수 규칙**을 엄격히 적용합니다.

1. **Repository**: `skn-ai22-251029/SKN22-4th-1Team`
2. **Assignees (담당자)**
   - **현재 터미널이나 시스템을 사용 중인 사용자 본인**으로 할당합니다.
   - MCP 환경에서는 현재 인증된 사용자, CLI 환경에서는 `gh api user -q ".login"`으로 사용자 ID를 동적으로 가져와 할당합니다.
3. **Labels (라벨)**
   - 문제의 심각도와 목적에 따라 다음 4가지 템플릿 라벨 중 **가장 적절한 1개 이상**을 반드시 포함합니다:
     - `긴급`: 크리티컬한 버그, 빌드 에러 등 즉각적인 조치가 필요한 경우
     - `상`: 주요 기능 개발, 핵심 로직 수정
     - `중`: 일반적인 기능 추가, UI 개선
     - `하`: 단순 리팩토링, 문서 수정, 오타 교정
4. **Issue Body (본문)**
   - 간단한 작업 요약, 변경 대상 파일, AI가 파악한 원인/목적을 마크다운으로 포함합니다.

### 🗂️ Projects 자동 등록 규칙
이슈가 생성되면 즉시 `SKN22-4th-1Team` 프로젝트(Project Board)에 등록합니다.

- **MCP 방식**: `mcp_github-mcp-server`의 Projects 관련 도구를 활용.
- **CLI 방식 (`gh`)**:
  1. 프로젝트 ID 찾기:
     ```bash
     gh project list --owner skn-ai22-251029 --format json --jq '.[] | select(.title=="SKN22-4th-1Team") | .id'
     ```
  2. 이슈 등록:
     ```bash
     gh project item-add <프로젝트ID> --owner skn-ai22-251029 --issue <방금생성한이슈번호>
     ```

## 🚨 주의사항
에이전트는 이슈를 무분별하게 중복 생성하지 않도록, 행동 전 관련 이슈가 이미 열려있는지 먼저 검색(`list_issues` 또는 `gh issue list`)해야 합니다.
